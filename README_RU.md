# PointCoordinates

[English version](README.md)

PointCoordinates — это плагин для меток на Minecraft-серверах, который помогает игрокам сохранять важные места, удобно хранить координаты и быстро возвращаться к ним.

Вместо того чтобы запоминать координаты или искать их в старых сообщениях чата, игроки могут создавать именованные метки, делиться открытыми точками, смотреть информацию о метке и телепортироваться к ней, если это разрешено на сервере. Для игроков это удобно, а для администратора — гибко и понятно в настройке.

## Почему это удобно

- Игрокам не нужно вручную хранить координаты
- Открытые метки позволяют делиться полезными местами
- Списки, поиск и информация о метке помогают быстро находить нужную точку
- Выключенные функции не засоряют интерфейс игроку
- Администратор может включать и отключать функции и управлять доступом через права

## Возможности для игроков

- Личные метки
- Открытые метки
- Информация о метке в чате
- Поиск по названию и по публичным тегам
- Переименование, перенос и удаление меток
- Телепортация к доступным меткам
- Списки меток с пагинацией
- Подтверждение важных действий

## Почему плагин удобно настраивать

- Личные и открытые метки настраиваются отдельно
- Телепортацию можно ограничивать по типу меток
- Алиасы команд можно включать и выключать по одному
- Help, usage и tab-complete подстраиваются под конфиг и права
- Файлы локализации обновляются без потери пользовательских значений
- Для хранилища меток автоматически создаются резервные копии
- Есть опциональные интеграции с популярными плагинами

## Команды

Основная команда:

- `/pcords`

Дополнительные алиасы:

- `/cords`
- `/pc`
- `/pt`

Каждый алиас можно отдельно включить или выключить в `config.yml`.

### Ветки команд

- `/pcords add <name>`
  Создаёт личную метку в текущей точке игрока.

- `/pcords open <name>`
  Создаёт открытую метку в текущей точке или делает существующую метку открытой.

- `/pcords list`
  Открывает списки меток. Какие именно режимы списка будут доступны, зависит от конфига и прав.

- `/pcords search`
  Открывает доступные варианты поиска. В зависимости от прав и настроек можно искать по названию или по публичным тегам.

- `/pcords info <name>`
  Показывает информацию о метке: координаты, мир, тип и дополнительные данные, если они включены.

- `/pcords tp <name>`
  Телепортирует к метке, если телепортация разрешена для этой метки и для этого игрока.

- `/pcords edit`
  Открывает доступные варианты редактирования. В зависимости от прав это может быть переименование, перенос или изменение тегов.

- `/pcords remove <name>`
  Удаляет метку.

- `/pcords reload`
  Перезагружает конфиг, локализации, алиасы и другие рабочие настройки плагина.

Если функция выключена или у игрока нет прав, она не будет отображаться в help, usage и tab-complete.

## Настройка

PointCoordinates можно оставить простым или, наоборот, сделать более насыщенным по возможностям.

Можно настроить:

- личные метки
- открытые метки
- телепортацию и кулдауны
- звуки и подтверждение действий
- поиск и публичные теги
- алиасы команд
- локализацию названий команд в help и usage
- дополнительные интеграции

Основные файлы:

- `config.yml` — настройки плагина
- `cords.yml` — сохранённые метки
- `languages/*.yml` — переводы

## Локализация

Встроенные языки:

- Английский
- Русский
- Испанский
- Французский
- Немецкий
- Бразильский португальский
- Упрощённый китайский

Локализация загружается так:

- сначала используется файл языка из папки плагина
- если в нём не хватает ключей, они добираются из встроенного файла внутри jar
- английский используется только как последний запасной вариант

При желании можно включить локализацию названий команд в help, usage и tab-complete.

## Права

Права разделены по действиям, чтобы администратор мог гибко настроить доступ.

### Базовый доступ

- `cords.use`
  Разрешает использовать основную команду плагина.

- `cords.reload`
  Разрешает перезагрузку плагина.

### Создание меток

- `cords.add`
  Разрешает создавать личные метки.

- `cords.open`
  Разрешает создавать открытые метки или переводить метку в открытую.

### Телепортация

- `cords.teleport.personal`
  Разрешает телепортироваться к личным меткам.

- `cords.teleport.public`
  Разрешает телепортироваться к открытым меткам.

- `cords.teleport.owned`
  Разрешает телепортироваться к своим меткам, даже если отдельное право на personal/public телепорт не выдано.

- `cords.teleport.bypass_disabled`
  Разрешает телепортироваться даже тогда, когда телепортация для этого типа меток отключена в конфиге.

- `cords.teleport.bypass_cooldown`
  Разрешает обходить кулдаун телепортации.

### Списки и информация

- `cords.list`
  Разрешает использовать команду списка меток.

- `cords.list.private`
  Разрешает смотреть списки личных меток.

- `cords.list.owned`
  Разрешает смотреть списки меток, принадлежащих игроку.

- `cords.list.open`
  Разрешает смотреть списки открытых меток.

- `cords.list.all`
  Разрешает смотреть объединённые списки всех доступных меток.

- `cords.info`
  Разрешает смотреть информацию о метке.

### Поиск

- `cords.search.name`
  Разрешает искать метки по названию.

- `cords.search.tag`
  Разрешает искать открытые метки по тегам.

### Редактирование

- `cords.edit.name`
  Разрешает переименовывать свои метки.

- `cords.edit.name.others`
  Разрешает переименовывать чужие метки.

- `cords.edit.move`
  Разрешает переносить свои метки в текущую точку.

- `cords.edit.move.others`
  Разрешает переносить чужие метки.

- `cords.edit.tag`
  Разрешает изменять теги у своих открытых меток.

- `cords.edit.tag.others`
  Разрешает изменять теги у чужих открытых меток.

### Удаление

- `cords.remove`
  Разрешает удалять свои метки.

- `cords.remove.others`
  Разрешает удалять чужие метки.

## Интеграции

Опционально поддерживаются:

- Vault
- CombatLogX
- Dynmap
- BlueMap

По умолчанию всё это выключено, пока ты сам не включишь нужные опции в конфиге.

## Примечания

- Имя плагина на сервере: `PointCoordinates`
- Основная команда: `/pcords`
- Алиасы можно включать и выключать по отдельности
- Игроки видят только тот функционал, который реально доступен им на сервере
