# PointCoordinates

[English version](README.md)

PointCoordinates — это плагин для меток на Minecraft-серверах, который помогает игрокам сохранять важные места, удобно хранить координаты и быстро возвращаться к ним.

Вместо того чтобы запоминать координаты или искать их в старых сообщениях чата, игроки могут создавать понятные именованные метки, делиться открытыми точками, смотреть информацию о метке и телепортироваться к ней, если это разрешено на сервере. Для игроков это просто и удобно, а для администратора — гибко и настраиваемо.

## Почему такой плагин удобен

- Игрокам не нужно вручную хранить координаты
- Открытые метки позволяют делиться полезными местами с другими
- Списки, поиск и информация о метке помогают быстро находить нужную точку
- Плагин не перегружает игроков лишними командами
- Администратор может включить только нужные функции и выключить всё остальное

## Возможности для игроков

- Личные метки
- Открытые метки
- Информация о метке в чате
- Поиск по названию и по публичным тегам
- Переименование, перенос и удаление меток
- Телепортация к доступным меткам
- Списки меток с пагинацией
- Подтверждение важных действий

## Почему плагин удобно настраивать

- Личные и открытые метки настраиваются отдельно
- Телепортацию можно ограничивать по типу меток
- Алиасы можно включать и выключать по одному
- Help, usage и tab-complete подстраиваются под конфиг и права
- Файлы локализации обновляются без потери пользовательских значений
- Для хранилища меток автоматически создаются резервные копии
- Есть опциональные интеграции с популярными плагинами

## Команды

Основная команда:

- `/pcords`

Дополнительные алиасы:

- `/cords`
- `/pc`
- `/pt`

Каждый алиас можно отдельно включить или выключить в `config.yml`.

Основные ветки команд:

- `/pcords add <name>` — создать личную метку
- `/pcords open <name>` — создать открытую метку или сделать свою метку открытой
- `/pcords list` — открыть список меток
- `/pcords search` — открыть варианты поиска
- `/pcords info <name>` — показать информацию о метке
- `/pcords tp <name>` — телепортироваться к метке
- `/pcords edit` — открыть варианты редактирования
- `/pcords remove <name>` — удалить метку
- `/pcords reload` — перезагрузить плагин

Если функция выключена или у игрока нет прав, она не будет отображаться в help, usage и tab-complete.

## Настройка

PointCoordinates можно оставить простым или, наоборот, сделать более насыщенным по возможностям.

Можно настроить:

- личные метки
- открытые метки
- телепортацию и кулдауны
- звуки и подтверждение действий
- поиск и публичные теги
- алиасы команд
- локализацию названий команд в help и usage
- дополнительные интеграции

Основные файлы:

- `config.yml` — настройки плагина
- `cords.yml` — сохранённые метки
- `languages/*.yml` — переводы

## Локализация

Встроенные языки:

- Английский
- Русский
- Испанский
- Французский
- Немецкий
- Бразильский португальский
- Упрощённый китайский

Локализация загружается так:

- сначала используется файл языка из папки плагина
- если в нём не хватает ключей, они добираются из встроенного файла внутри jar
- английский используется только как последний запасной вариант

При желании можно включить локализацию названий команд в help, usage и tab-complete.

## Права

Права разделены по действиям, чтобы администратор мог гибко настроить доступ.

Примеры:

- `cords.add`
- `cords.open`
- `cords.info`
- `cords.search.name`
- `cords.search.tag`
- `cords.edit.name`
- `cords.edit.move`
- `cords.edit.tag`
- `cords.remove`
- `cords.teleport.personal`
- `cords.teleport.public`
- `cords.teleport.owned`
- `cords.reload`

Для действий с чужими метками есть отдельные права вида `*.others`.

## Интеграции

Опционально поддерживаются:

- Vault
- CombatLogX
- Dynmap
- BlueMap

По умолчанию всё это выключено, пока ты сам не включишь нужные опции в конфиге.

## Примечания

- Имя плагина на сервере: `PointCoordinates`
- Основная команда: `/pcords`
- Алиасы можно включать и выключать по отдельности
- Игроки видят только тот функционал, который реально доступен им на сервере
